# Operations

The operations domain owns procedures: repository management, publishing, audits, and runbooks.

## Current Areas

- [Repository Operations](repository/index.md)
- [Publishing](publishing/index.md)
- [Audits](audits/index.md)

## Ownership

Primary owner: Product Operations / Engineering Operations  
Required reviewers when changed: The person accountable for executing the runbook.

## Belongs Here

- GitHub setup and repo split procedures.
- External publishing steps.
- Completion audits.
- Operational checklists.

## Does Not Belong Here

- Durable architecture decisions.
- Product requirements.
- Raw source material.
